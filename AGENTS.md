# AGENTS

## Repo Shape
- This repo is a prose workspace, not a software project. There are no verified build, test, lint, formatter, CI, or package-manager files in the root.
- `1.txt` through `9.txt` are episode/script draft files. Treat the numbered `.txt` files as the main manuscript artifacts.
- `synopsis.md` is the highest-level end-to-end story spine for `85%`.
- `primer.md` is the main science/worldbuilding reference and contains the most detailed physics constraints.
- `ideas.md`, `suggestions.md`, and `dark_matter_convo.md` are supporting development notes. Use them as idea banks, not as higher-priority canon than `synopsis.md` plus the episode drafts.

## Working Priorities
- When making story edits, check `synopsis.md` first for plot-level intent, then the relevant episode `.txt`, then `primer.md` for physics consistency.
- If notes conflict, prefer the concrete manuscript (`synopsis.md` and numbered episode files) over brainstorming/advisory files.
- Preserve the story's established sensory grammar: sub-bass hums, gravity distortions, silent catastrophic events, and physics-first descriptions recur across the manuscript.

## Editing Guidance
- Keep terminology consistent with the existing manuscript: `Dark Sector`, `X17 Polymer`, `Dark Photon`, `Planet Nine`, `The Forge`, `Cold Burn`, `Newton Sanction`.
- Match the current file roles before restructuring anything: markdown files hold canon/notes; numbered `.txt` files hold episode prose.
- Do not invent nonexistent repo workflows in future instructions. If you mention verification, state plainly that no automated checks are present in this repo.
